# Picot 国际化 (i18n) 设计文档

> **日期**: 2026-07-08
> **状态**: 设计已批准（含两轮 code review 修订），待实施
> **作者**: Dr. Lin & Mr. Spock

## 1. 目标

为 Picot 添加国际化支持，首期支持**中文**和**英文**，架构上预留扩展其他语言的能力。

**核心需求：**
1. 所有页面的用户可见文本支持 i18n
2. Settings → General 新增"Language"设置，允许用户手动选择：System Default / 中文 / English
3. 语言偏好跨 workspace 窗口共享（每个 workspace 运行在不同 `localhost:port`）
4. 语言切换后**即时生效**：静态 DOM 文本和已渲染的动态 JS 组件文本都要更新

## 2. 约束

| 约束 | 说明 |
|------|------|
| 纯原生 JS，无框架 | i18n 引擎必须零依赖、零构建步骤 |
| 无前端构建管线 | ES modules 直接以 `<script type=module>` 加载，不能引入打包工具 |
| Cookie 持久化（非 localStorage） | 每个 workspace 是不同端口的 `localhost:300X`，localStorage 按端口隔离。语言偏好必须跨端口共享——与 `themes.js` 完全同样的约束和模式 |
| Tauri v2 桌面应用 | 非 Web 部署，无 SSR |
| `cost.html` 在 iframe 中加载 | 同源同端口，共享 cookie，但 iframe 无法监听父页面 cookie 变化，需 postMessage 同步 |
| `bootstrap.html` 是独立错误页 | 纯静态 `<script>`，无 ES module，需特殊处理 |
| Picot 大量文本是 JS 运行时动态生成 | `applyTranslations()` 只处理当前 DOM 上已有的 `data-i18n` 属性节点；JS 生成的文本需 locale-change 事件驱动重新渲染 |
| `t()` 返回纯文本，不可直接插入 innerHTML | 翻译字符串和插值参数进入 innerHTML 前必须 `escapeHtml()`，或用 DOM 创建方式 |

## 3. 方案选择

**选定方案：自建轻量 i18n 模块**（放弃 i18next / 多份 HTML 预渲染方案）

**理由：**
- Picot 的 i18n 需求极简：~500 个扁平字符串 + `{var}` 插值，不需要复数、延迟加载、命名空间等高级功能
- 自建引擎 ~250 行代码，与 `themes.js` 的 cookie 模式完全对称
- 零依赖、零构建，符合项目核心约束
- 后续加语言只需加一个 JSON 文件

## 4. 架构设计

### 4.1 文件结构

```
public/
  i18n.js                      ← 新模块：i18n 引擎（核心）
  locales/
    en.json                    ← 英文翻译（source of truth + fallback）
    zh.json                    ← 中文翻译
  index.html                   ← 静态文本加 data-i18n 属性
  cost.html                    ← 静态文本加 data-i18n 属性（Phase 2）
  bootstrap.html               ← 内联翻译逻辑
  app.js                       ← 引入 i18n，await initI18n()，动态文本改 t() 调用
  各模块 .js                    ← 动态文本改 t() 调用，订阅 onLocaleChange
  app-voice-input.js           ← recognition.lang 跟随 locale
```

### 4.2 i18n 引擎设计 (`public/i18n.js`)

#### 导出 API

```javascript
// 初始化：加载 en.json（fallback）+ 当前 locale JSON，扫描 DOM
// 内部捕获所有错误，失败时 fallback 英文，绝不阻塞 app 启动
export async function initI18n(): Promise<void>;

// 翻译函数：支持 {var} 插值，英文 fallback
// 返回纯文本——调用方不可直接插入 innerHTML，必须先 escapeHtml()
export function t(key: string, params?: Record<string, string|number>): string;

// 设置语言：请求 locale 加载成功后才写 cookie，重新扫描 DOM，广播 locale-change 事件
// 内含 sequence guard 防止快速切换时旧 locale 覆盖新选择；加载失败时回退到英文但不持久化坏偏好
export async function setLocale(preference: string): Promise<void>;

// 获取当前有效语言代码（"en" | "zh"）
export function getLocale(): string;

// 获取用户选择（"system" | "en" | "zh"），用于设置面板高亮
// 非法 cookie 值归一化为 "system"
export function getLanguagePreference(): string;

// 订阅 locale 变化：动态组件注册回调，在语言切换时重新渲染
// 返回 unsubscribe 函数；临时组件必须在关闭时调用
export function onLocaleChange(listener: (detail: { locale: string; preference: string }) => void): () => void;

// 语言选项常量（设置面板用）
export const LANGUAGES: Array<{ value: string; labelKey?: string; nativeLabel?: string }>;

// 纯函数：从偏好解析 locale，供测试注入
export function resolveLocale(preference: string, systemLanguage?: string): string;
```

#### 核心状态

```javascript
const SUPPORTED_PREFERENCES = new Set(["system", "en", "zh"]);

const BCP47_TAG = { en: "en", zh: "zh-CN" };

let enMessages = {};           // 英文 messages，初始化时加载一次，永不卸载
let activeMessages = {};       // 当前 locale 的 messages
let currentLocale = "en";      // 解析后的 locale ("en" | "zh")
let currentPreference = "system"; // 用户偏好 ("system" | "en" | "zh")
let localeLoadSequence = 0;   // 竞态保护计数器
const listeners = new Set();   // onLocaleChange 注册的回调
```

#### 偏好归一化

```javascript
function normalizePreference(preference) {
  return SUPPORTED_PREFERENCES.has(preference) ? preference : "system";
}
```

非法 cookie 值（如 "fr"、undefined、旧版本残留）归一化为 `"system"`。不重新持久化非法值。

#### Locale 解析

```javascript
export function resolveLocale(preference, systemLanguage) {
  const lang = systemLanguage ?? (typeof navigator !== "undefined" ? navigator.language : "en");
  const normalized = normalizePreference(preference);
  if (normalized === "zh") return "zh";
  if (normalized === "en") return "en";
  // system: 前缀匹配
  return lang?.toLowerCase().startsWith("zh") ? "zh" : "en";
}
```

#### 初始化

```
initI18n()  ← await，阻塞主 app 初始化；但内部捕获所有错误，失败时 fallback 英文
  ├── requestedPreference = normalizePreference(readLanguageCookie() ?? "system")
  ├── requestedLocale = resolveLocale(requestedPreference, navigator.language)
  ├── try:
  │     ├── enMessages = await fetchLocale("en")        ← 始终加载英文作为 fallback
  │     └── result = await loadMessagesForLocale(requestedLocale)
  │           ├── 成功：{ locale: requestedLocale, messages, fallback: false }
  │           └── 失败：{ locale: "en", messages: enMessages, fallback: true, failedLocale }
  │   catch:
  │     ├── enMessages = {} (或上次缓存的值)
  │     └── result = { locale: "en", messages: enMessages, fallback: true }
  │     ← 不抛错，app 以英文（或 key 本身）继续启动
  ├── activeMessages = result.messages
  ├── currentPreference = requestedPreference          ← 用户选择；可与实际 locale 不同
  ├── currentLocale = result.locale                    ← 实际生效 locale；fallback 后为 "en"
  ├── document.documentElement.lang = BCP47_TAG[currentLocale] ?? "en"
  └── applyTranslations(document)
```

**语义边界**：
- `getLanguagePreference()` 返回用户偏好：`"system"` / `"en"` / `"zh"`。
- `getLocale()` 返回实际生效 locale：`"en"` / `"zh"`。如果请求的 locale 加载失败并 fallback 到英文，`getLocale()` 返回 `"en"`。
- `document.documentElement.lang` 必须跟实际文本语言一致，使用 `currentLocale`，不能使用请求的 locale。

#### fetch error policy

`loadMessagesForLocale()` 返回结构化结果，不吞掉失败状态。这样 `setLocale()` 能区分“请求 locale 成功”与“fallback 到英文”，避免把坏偏好写入 cookie。

```javascript
async function loadMessagesForLocale(locale) {
  if (locale === "en") {
    return { locale: "en", messages: enMessages, fallback: false };
  }

  try {
    return {
      locale,
      messages: await fetchLocale(locale),
      fallback: false,
    };
  } catch (error) {
    console.warn(`[i18n] Failed to load ${locale}, falling back to English`, error);
    return {
      locale: "en",
      messages: enMessages,
      fallback: true,
      failedLocale: locale,
    };
  }
}
```

#### setLocale（含竞态保护 + fetch-then-persist）

```
setLocale(preference)
  ├── requestedPreference = normalizePreference(preference)
  ├── requestedLocale = resolveLocale(requestedPreference, navigator.language)
  ├── sequence = ++localeLoadSequence               ← 竞态保护
  ├── result = await loadMessagesForLocale(requestedLocale)
  ├── if (sequence !== localeLoadSequence) return   ← 旧请求晚返回，丢弃
  ├── activeMessages = result.messages
  ├── currentLocale = result.locale                 ← 实际生效 locale；fallback 后为 "en"
  ├── if result.fallback:
  │     ├── 不写 cookie
  │     ├── 不把 currentPreference 改成失败的 requestedPreference
  │     └── 保留上一次有效 preference（或初始化时的 normalized preference）
  ├── else:
  │     ├── currentPreference = requestedPreference
  │     └── writeLanguageCookie(requestedPreference) ← 请求 locale 成功后才持久化
  ├── document.documentElement.lang = BCP47_TAG[currentLocale] ?? "en"
  ├── applyTranslations(document)                    ← 更新静态 DOM data-i18n 节点
  └── broadcastLocaleChange(currentLocale, currentPreference)
        ├── 遍历 listeners，调用每个回调
        └── window.dispatchEvent(new CustomEvent("picot:locale-change", { detail }))
```

**竞态保护**：用户快速切换语言时，晚返回的旧 locale fetch 不会覆盖新选择。`localeLoadSequence` 是单调递增计数器，与 `FileBrowser.loadSequence` 同一模式。

**fetch-then-persist**：只有请求的 locale 文件成功加载时才写 cookie。若请求 locale 失败，当前运行时 fallback 到英文，`getLocale()` 返回 `"en"`，`document.lang` 为 `"en"`，但不会把失败的 preference 写入 cookie。

#### t() 翻译函数

```
t(key, params)
  ├── translated = lookup(activeMessages, key)
  ├── if translated != null → interpolate({var}) → 返回
  ├── fallback = lookup(enMessages, key)
  ├── if fallback != null → interpolate({var}) → 返回
  ├── console.warn(`[i18n] Missing key: ${key}`)
  └── 返回 key 本身
```

**安全约束**：`t()` 返回纯文本。调用方**不可**直接插入 `innerHTML` / `insertAdjacentHTML` / template string。必须：
- 写入 `textContent` / `placeholder` / `title` / `aria-label`，或
- 经过 `escapeHtml(t(...))` 后才能进入 `innerHTML`

```javascript
// ✅ 正确
el.textContent = t("files.loading");
el.innerHTML = `<div class="file-loading">${escapeHtml(t("files.loading"))}</div>`;

// ❌ 错误——禁止
el.innerHTML = `<div class="file-loading">${t("files.loading")}</div>`;
```

Code review 时 grep：`innerHTML.*t\(`、`insertAdjacentHTML.*t\(`、template string `${t(`，只允许搭配 `escapeHtml()`。

#### Cookie 策略

与 `themes.js` 完全一致的实现模式（encode/decode、SameSite=Lax、try/catch、Biome ignore）：

```javascript
const LANG_COOKIE = "picot-language";
const LANG_COOKIE_MAX_AGE_SECONDS = 60 * 60 * 24 * 365 * 10; // 10 年

function readLanguageCookie() {
  try {
    const cookies = document.cookie ? document.cookie.split("; ") : [];
    for (const entry of cookies) {
      const eq = entry.indexOf("=");
      if (eq === -1) continue;
      if (entry.slice(0, eq) !== LANG_COOKIE) continue;
      const raw = entry.slice(eq + 1);
      try {
        return decodeURIComponent(raw);
      } catch {
        return raw;
      }
    }
  } catch {
    // document.cookie can throw in sandboxed contexts; treat as missing.
  }
  return null;
}

function writeLanguageCookie(preference) {
  try {
    const value = encodeURIComponent(preference);
    // biome-ignore lint/suspicious/noDocumentCookie: Cookie Store API is async and not suitable for synchronous cross-port language persistence
    document.cookie = `${LANG_COOKIE}=${value}; Max-Age=${LANG_COOKIE_MAX_AGE_SECONDS}; Path=/; SameSite=Lax`;
  } catch {
    // ignore — same fallback as the read path
  }
}
```

Cookie 值存的是用户偏好（`"system"` / `"en"` / `"zh"`），而非解析后的 locale。

#### onLocaleChange 生命周期

```javascript
export function onLocaleChange(listener) {
  listeners.add(listener);
  return () => listeners.delete(listener);
}
```

**生命周期规则**：
- **单例模块**（SessionSidebar、MessageRenderer、FileBrowser、app-updater、DialogHandler 等）：构造时注册一次，listener 保留整个 app 生命周期；回调内部检查当前 UI 是否存在。
- **临时组件**（每次打开都新建实例的 FolderPicker 等）：必须存储 unsubscribe 函数，在 `destroy()` / `close()` 时调用。
- 不要在单例的普通 `close()` 方法里 unsubscribe；否则下一次打开同一单例 UI 时将失去 locale-change 支持。

```javascript
// 临时组件示例
class FolderPicker {
  constructor(...) {
    this.unsubscribeLocaleChange = onLocaleChange(() => {
      if (this.isOpen) this.renderLabels();
    });
  }

  destroy() {
    this.unsubscribeLocaleChange?.();
  }
}
```

#### applyTranslations

```
applyTranslations(root)
  ├── 扫描 [data-i18n]             → element.textContent = t(key)
  ├── 扫描 [data-i18n-ph]          → element.placeholder = t(key)
  ├── 扫描 [data-i18n-title]       → element.title = t(key)
  ├── 扫描 [data-i18n-aria-label]  → element.setAttribute("aria-label", t(key))
  └── 注意：所有写入 textContent，不涉及 innerHTML
```

**不支持 `data-i18n-html`**（Phase 1）。需要 HTML 结构的复杂文本用拆分 DOM 节点解决（见 4.5）。

### 4.3 Key 命名规范

- **点分嵌套**，按模块/功能区域分组：`sidebar.settings`、`composer.placeholder`
- 第一级对应模块名（`sidebar`、`composer`、`settings`、`files`、`status`、`errors`、`actions`、`messages`、`tools`、`onboarding`、`bootstrap`）
- **actions.*** 存按钮文本（Save、Cancel、Remove、Retry）
- **status.*** 存状态文本（Saving、Saved、Connecting、Connected 等）
- **errors.*** 只存错误消息，不放按钮文本
- 避免超过 3 层嵌套

### 4.4 HTML 标注约定

| 属性 | 替换目标 | 示例 |
|------|---------|------|
| `data-i18n="key"` | `textContent` | `<span data-i18n="sidebar.settings">Settings</span>` |
| `data-i18n-ph="key"` | `placeholder` | `<input data-i18n-ph="sidebar.search" placeholder="Search...">` |
| `data-i18n-title="key"` | `title` | `<button data-i18n-title="sidebar.openFolder" title="Open folder">` |
| `data-i18n-aria-label="key"` | `aria-label` | `<button data-i18n-aria-label="sidebar.refresh" aria-label="Refresh">` |

**原则**：HTML 中保留英文原文作为 fallback（在 i18n 初始化前的极短时间内可见），避免空文本闪烁。

**不支持 `data-i18n-html`**（Phase 1）。需要 HTML 结构的复杂文本用拆分 DOM 节点解决（见 4.5）。

### 4.5 复杂 HTML 文本的拆分方案

不含 `data-i18n-html`，需要 HTML 结构的文本（如 settings help 段落中的 `<code>` 和 `<a>` 标签）用拆分 DOM 节点解决：

### 4.6 语言选择器 UI

在 Settings → General → Appearance 下方新增 **Language** section：

```html
<div class="settings-section">
  <div class="settings-section-title" data-i18n="settings.language.title">Language</div>
  <div class="settings-language-options" id="settings-language-options">
    <!-- 由 JS 动态生成 -->
  </div>
  <p class="settings-help" data-i18n="settings.language.description">
    Choose the interface language. Changes apply immediately.
  </p>
</div>
```

语言选项常量：

```javascript
const LANGUAGES = [
  { value: "system", labelKey: "settings.language.systemDefault" },
  { value: "en", nativeLabel: "English" },
  { value: "zh", nativeLabel: "中文" },
];
```

渲染逻辑（参考 `buildThemeGrid()` 的模式）：

```javascript
function buildLanguageSelector() {
  const container = document.getElementById("settings-language-options");
  const current = getLanguagePreference();
  container.innerHTML = "";
  for (const lang of LANGUAGES) {
    const label = document.createElement("label");
    label.className = "settings-language-option";
    const radio = document.createElement("input");
    radio.type = "radio";
    radio.name = "language";
    radio.value = lang.value;
    radio.checked = current === lang.value;
    radio.addEventListener("change", () => setLocale(lang.value));
    const span = document.createElement("span");
    // 语言名称用本语种显示（English / 中文）；System Default 跟随 UI 翻译
    span.textContent = lang.nativeLabel ?? t(lang.labelKey);
    label.appendChild(radio);
    label.appendChild(span);
    container.appendChild(label);
  }
}
```

`buildLanguageSelector()` 也注册 `onLocaleChange`，因为 "System default" label 跟随当前 UI 语言变化。

**设计决策**：
- 语言名称（English / 中文）始终用本语种显示——用户需要认出自己要选的语言
- "System default" 跟随当前 UI 语言翻译，否则中文界面里唯一英文残留就是它

### 4.7 初始化时序

`app.js` 中的初始化顺序，`await initI18n()` 放在 `applyTheme()` 之后、其他 UI 初始化之前：

```javascript
// app.js 初始化区域（约 line 3552）

// Restore saved theme
const savedTheme = getCurrentTheme();
applyTheme(savedTheme);

// Initialize i18n — 阻塞直到 locale 加载完成（内部 catch，不阻塞 app 启动）
await initI18n();

// 以下初始化在 locale ready 后执行
setupContextViz({ ... });
setupVoiceInput({ ... });
buildThemeGrid();
buildLanguageSelector();
```

Tauri v2 桌面 WebView 支持 ES module top-level await。locale JSON < 15KB，阻塞几十毫秒可接受。若不想用 top-level await，包一层 `async function main() { ... } main();`。

`initI18n()` 内部捕获所有错误：locale 文件加载失败时 fallback 到英文，`enMessages` 为空时 `t()` 返回 key 本身。**最差情况显示英文或 key，绝不阻止 Picot 启动。**

### 4.8 动态文本改造模式

#### 模式 1：DOM 创建（首选）

```javascript
// 改造前
this.container.innerHTML = '<div class="file-loading">Loading…</div>';

// 改造后——DOM 创建，无 innerHTML
this.container.replaceChildren();
const loading = document.createElement("div");
loading.className = "file-loading";
loading.textContent = t("files.loading");
this.container.appendChild(loading);
```

#### 模式 2：innerHTML + escapeHtml（现有模板代码不便重构时）

```javascript
// 改造前
this.container.innerHTML = `<div class="file-loading">Loading…</div>`;

// 改造后——t() 输出必须 escapeHtml
this.container.innerHTML = `<div class="file-loading">${escapeHtml(t("files.loading"))}</div>`;
```

> **注意**：Picot 当前有三个 `escapeHtml` 实现（`app.js`、`cost-infobar.js`、`markdown.js`）。改造时使用所在模块已有的 `escapeHtml`，或从 `markdown.js` 导入。

#### 模式 3：带插值的错误消息

```javascript
// 改造前
renderError(`Failed to start new session: ${e}`);

// 改造后——error 参数来自底层；renderError 必须用 textContent，或在内部 escapeHtml(message)
const message = t("errors.failedToStart", { what: "session", error: String(e) });
renderError(message);
// 禁止把 message 未转义地拼进 innerHTML。
```

**底层错误边界**：`transport.js` 等底层模块的协议错误不翻译。上层展示时用 `t()` 包裹用户可见的前缀文本，原始 error message 保留英文作为 `{error}` 插值参数。

`setLocale()` 广播事件后，各动态组件的回调负责重新渲染自身文本。这是"即时切换"成立的核心机制：

| 组件 | onLocaleChange 回调 | 即时更新范围 |
|------|-------------------|-------------|
| `buildLanguageSelector()` | 重建选项列表 | "System default" label 跟随 locale |
| `SessionSidebar` | `this.render()`，保留 scrollTop | 全部 sidebar 文本 |
| `MessageRenderer` | 若 welcome 可见则重渲染 welcome；扫描已渲染消息按钮；更新 thinking labels | welcome + thinking + copy message aria-label/title |
| `Markdown` | 扫描已渲染代码块 copy 按钮 | Copy/Copied 按钮文本 |
| `FileBrowser` | 重绘 loading/empty/error 文本；文件名不翻译 | loading/empty/error |
| `FolderPicker` | 若弹窗打开则重绘文本 | 全部弹窗文本 |
| `app-updater` | 重新应用当前 updater status | Checking/Download/Retry 等 |
| `app-voice-input` | 更新 `recognition.lang` | 下次录音生效 |
| `app-settings-editors` | 若 API key panel 打开则重绘 loading/error 文本 | loading/error |

### 4.12 "System Default" 语义

- "System Default" 在**每次窗口初始化**时解析 `navigator.language`：`"zh*"` 前缀 → `"zh"`，其余 → `"en"`
- **不承诺** app 运行中监听 OS language change 事件（WebView 不可靠触发 `navigator.language` 变化）
- 若用户保持窗口打开并修改系统语言，需重新打开窗口或重新选择 "System Default" 才生效
- 设置面板文案：

> System Default resolves from your system language at app startup. Runtime OS language changes are picked up after reopening the window.

### 4.13 Cost 页面（iframe）i18n（Phase 2）

`cost.html` 在 iframe 中加载，同源同端口，共享 cookie。初始化时调用 `await initI18n()` 即可。但父页面 Settings 中切换语言时，iframe 无法监听 cookie 变化。

**同步协议**：父页面 `setLocale()` 完成后通过 `postMessage` 通知 iframe：

```javascript
// app.js 中 setLocale 完成后
const costIframe = document.querySelector(".settings-usage-iframe");
costIframe?.contentWindow?.postMessage(
  { type: "picot:locale-change", preference: getLanguagePreference() },
  window.location.origin,
);
```

```javascript
// cost.js — 使用 applyLocalePreference 而非 initI18n，带竞态保护
window.addEventListener("message", async (event) => {
  if (event.origin !== window.location.origin) return;
  if (event.data?.type !== "picot:locale-change") return;
  await applyLocalePreference(event.data.preference);
});
```

`applyLocalePreference(preference)` 复用 `setLocale` 的竞态保护逻辑（`localeLoadSequence`），不重复加载已缓存的 `enMessages`，错误时 fallback 英文。

### 4.14 Bootstrap 错误页 i18n

`bootstrap.html` 是独立错误页，使用内联 `<script>`（非 ES module），只有约 8 个字符串。

**关键**：bootstrap 必须正确处理 `"system"` 偏好——cookie 存的是 preference 不是 resolved locale。中文系统 + cookie=system 必须显示中文。

```javascript
// bootstrap.html 内联 script
function readLanguagePreference() {
  try {
    const cookies = document.cookie ? document.cookie.split("; ") : [];
    for (const entry of cookies) {
      const eq = entry.indexOf("=");
      if (eq === -1) continue;
      if (entry.slice(0, eq) !== "picot-language") continue;
      const raw = entry.slice(eq + 1);
      try { return decodeURIComponent(raw); } catch { return raw; }
    }
  } catch {}
  return "system";
}

function resolveBootstrapLocale(preference) {
  if (preference === "zh") return "zh";
  if (preference === "en") return "en";
  // system: 前缀匹配 navigator.language
  return navigator.language?.toLowerCase().startsWith("zh") ? "zh" : "en";
}

const preference = readLanguagePreference();
const locale = resolveBootstrapLocale(preference);
const messages = locale === "zh" ? zhMessages : enMessages;
document.documentElement.lang = locale === "zh" ? "zh-CN" : "en";
// 替换各元素文本
```

8 个字符串的翻译直接内联在 script 中，不单独建 locale JSON。`zhMessages` 和 `enMessages` 是内联对象常量。

## 7. 风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| 翻译遗漏（某 key 只在 en.json 没有 zh.json） | 高 | `t()` 自动 fallback 到英文（`enMessages` 常驻）；CI key 完整性测试强制 |
| 动态文本遗漏 i18n 改造 | 中 | 逐模块 checklist（4.11 表），code review 对照 |
| locale 切换后动态组件未重渲染 | 高 | `onLocaleChange` 事件机制 + 每个动态组件必须订阅；集成验证逐组件检查 |
| 快速切换导致旧 locale 覆盖新选择 | 高 | `localeLoadSequence` 竞态保护（同 FileBrowser 模式） |
| `t()` 输出注入 innerHTML | 高 | 硬约束：必须 `escapeHtml(t(...))` 或用 DOM 创建；code review grep `innerHTML.*t\(` |
| locale 文件加载失败 | 低 | `loadMessagesForLocale` 返回结构化 fallback 结果；`initI18n()` 不阻塞 app 启动 |
| fetch 失败后 cookie 持久化坏偏好 | 中 | fetch-then-persist：请求 locale 成功加载后才写 cookie；fallback 到英文时不写坏偏好 |
| bootstrap system 解析错误 | 中 | `resolveBootstrapLocale` 正确处理 "system" 偏好 + `decodeURIComponent` |
| 翻译质量 | 中 | 中文翻译由 Dr. Lin 审校 |
| `app.js` 122KB 改动冲突 | 中 | i18n 改动以 `t()` 替换为主，改动点小而散，降低合并冲突 |
| `onLocaleChange` 回调泄漏（临时组件） | 中 | 临时组件存储 unsubscribe，destroy/close 时调用 |

## 8. 非目标 (Non-Goals)

- **不做** RTL（从右到左）语言支持

- **不做** 按需加载 locale 文件（两种语言总共 ~30KB，不值得懒加载）
- **不做** 服务端翻译（pi 的 RPC 响应不翻译——那是 pi 的职责，Picot 只管自己的 UI）
- **不做** `data-i18n-html`（Phase 1，需要 HTML 结构的文本用拆分 DOM 节点解决）
- **不做** 运行时 OS language change 监听（WebView 不可靠，重新打开窗口时跟随即可）
